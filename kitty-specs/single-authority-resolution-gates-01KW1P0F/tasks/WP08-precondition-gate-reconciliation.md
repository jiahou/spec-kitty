### T038 — Reconcile existing surface-resolver gate floors + allowlist

After WP02–WP05 have landed on the branch, the production source tree has changed: call sites
that were raw-join bypasses may now be routed through the canonical seam, and others may have
moved lines. The existing gate in `test_single_mission_surface_resolver.py` uses:

- `_MIN_DISCOVERED_ROWS: int = 20` — the anti-vacuous floor
- `_RAW_JOIN_SITES` — a tuple of `(rel_path, seed_line, rationale)` triples used to derive
  composite keys for allowlisted bypass sites

**Reconciliation steps:**

1. Run `discover_rows()` (from `surface_resolution_audit/audit.py`) against the post-sweep
   `src/` tree and count the returned rows.
2. If the count has changed from the pre-sweep value, update `_MIN_DISCOVERED_ROWS` to the
   new minimum that is anti-vacuous (use `new_count - 2` as a safe floor, never `0` or `1`).
3. For each entry in `_RAW_JOIN_SITES`, re-verify that the `(rel_path, seed_line)` still
   resolves to a live composite key matching the same allowlisted join. Use the
   `composite_key_from_file` helper (already imported at line 113 of the existing gate) to
   re-derive each key. If a seed line has drifted (the join moved due to WP02–WP05 edits),
   update the seed line to its new position.
4. If any allowlisted site was ROUTED by WP02–WP05 (no longer a raw join), remove it from
   `_RAW_JOIN_SITES`. The allowlist must be shrink-only per NFR-003.
5. Run the full gate test to confirm green: `pytest tests/architectural/test_single_mission_surface_resolver.py -q`.

**Out-of-map edits**: if `surface_resolution_audit/audit.py` itself needs an update (e.g., a
site name changed in `_RESOLVER_SOURCE_STEMS`), this is an out-of-WP01-map edit. Record it
explicitly in the commit message: "Out-of-map edit to surface_resolution_audit/audit.py:
<rationale>". Do NOT skip the edit — but document it; the no-overlap rule (CLAUDE.md) is a
communication obligation, not a prohibition on necessary edits.

**Gate correctness invariants** (from the existing test docstring):
- The walker runs live on the current source tree (not a static inventory).
- Composite keys are derived via `composite_key_from_file`, never hand-authored.
- The anti-vacuous floor is a concrete integer `> 1`, never `0`.
- A staleness test (`test_allowlist_entries_are_not_stale`) must still pass — if an allowlisted
  entry no longer matches a live site, the gate will fail on this test; that is the signal to
  remove the entry.

**GAP-5 — No silent downward floor:** If the reconciled `_MIN_DISCOVERED_ROWS` ends up LOWER than its pre-sweep value of `20`, you MUST record in the commit body the exact delta and the specific routed sites that account for it (i.e. which formerly raw-join sites are now canonical seam calls and thus no longer appear in `discover_rows()` output). An UNEXPLAINED downward floor move — setting `_MIN_DISCOVERED_ROWS` to a lower number without naming which sites disappeared — is a merge-blocker. The floor may legitimately decrease if sites were routed (fewer raw-join bypasses to discover), but the accounting must be explicit and traceable.
