## Objective

Complete the Claude Code plugin with: (1) `bin/spec-kitty-wrapper` runtime bootstrap scripts (bash + Windows CMD) with PATH-check and `uvx` fallback; (2) a CI validation workflow; (3) `marketplace.json` for git-based plugin distribution; (4) developer documentation. ## Context

The plugin bundle structure is produced by WP04. This WP adds the runtime layer and distribution metadata. Key research findings (research.md R-01, quickstart.md):

- Claude Code plugins are installed via `claude plugin install <url>` (marketplace) or `claude --plugin-dir <path>` (dev mode)
- The `bin/` wrapper is the runtime bootstrap that Claude Code calls when executing a skill
- Two deployment scenarios: (A) user already has `spec-kitty` on PATH — wrapper delegates directly; (B) user only has `uvx` — wrapper falls back to `uvx spec-kitty-cli==<version>`
- `marketplace.json` enables `claude plugin marketplace add <repo-url>` for git-based installs
- The CI job must install the `claude` CLI, build the plugin, and run `validate --strict`

Refer to `contracts/plugin-manifest-claude.md` for the `marketplace.json` schema.

## Subtask Guidance

### T020 — Generate `bin/spec-kitty-wrapper` (bash) + `spec-kitty-wrapper.cmd` (Windows)

The wrapper scripts are generated into the bundle at build time (inside `ClaudeBundleProjector._copy_skills()` or a new `_write_wrappers()` step). They are NOT static files committed to the repo — the version is substituted at build time.

**bash wrapper** (`bin/spec-kitty-wrapper`):
```bash
#!/usr/bin/env bash
# Spec Kitty runtime bootstrap — PATH-check with uvx fallback.
# Generated by spec-kitty plugin build. Do not edit manually.
set -euo pipefail

VERSION="__SPEC_KITTY_VERSION__"  # substituted at build time

if command -v spec-kitty >/dev/null 2>&1; then
    exec spec-kitty "$@"
fi

if command -v uvx >/dev/null 2>&1; then
    exec uvx "spec-kitty-cli==${VERSION}" "$@"
fi

echo "Error: spec-kitty is not installed and uvx is not available." >&2
echo "Install spec-kitty: pip install spec-kitty-cli" >&2
exit 1
```

**Windows CMD wrapper** (`bin/spec-kitty-wrapper.cmd`):
```batch
@echo off
REM Spec Kitty runtime bootstrap for Windows. Generated by spec-kitty plugin build.
SET VERSION=__SPEC_KITTY_VERSION__

where spec-kitty >nul 2>&1
IF NOT ERRORLEVEL 1 (
    spec-kitty %*
    EXIT /B %ERRORLEVEL%
)

where uvx >nul 2>&1
IF NOT ERRORLEVEL 1 (
    uvx spec-kitty-cli==%VERSION% %*
    EXIT /B %ERRORLEVEL%
)

echo Error: spec-kitty is not installed and uvx is not available.
echo Install spec-kitty: pip install spec-kitty-cli
EXIT /B 1
```

Generate both in `ClaudeBundleProjector._write_wrappers(bundle_dir, version)`. Substitute `__SPEC_KITTY_VERSION__` with the actual version string. Set the bash wrapper executable (`chmod 0o755`) via Python's `os.chmod`.

Add `"bin": "bin/spec-kitty-wrapper"` (and `"binWindows": "bin/spec-kitty-wrapper.cmd"`) to `plugin.json` if the Claude plugin spec supports binary pointers. If it does not, omit and document that the wrapper is invoked via skill command substitution only.

### T021 — Add `.github/workflows/plugin-validate.yml` CI job

Create `.github/workflows/plugin-validate.yml`:

```yaml
name: Plugin Validate

on:
  push:
    branches: [main, "feat/**"]
    paths:
      - "src/specify_cli/tool_surface/bundles/**"
      - "src/specify_cli/cli/commands/plugin.py"
      - ".github/workflows/plugin-validate.yml"
  pull_request:
    paths:
      - "src/specify_cli/tool_surface/bundles/**"
      - "src/specify_cli/cli/commands/plugin.py"

jobs:
  validate-claude-plugin:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
      - name: Install Claude CLI
        run: npm install -g @anthropic-ai/claude-code
      - uses: astral-sh/setup-uv@v5
      - name: Install spec-kitty-cli
        run: uv sync --frozen --all-extras
      - name: Build Claude Code plugin
        run: uv run spec-kitty plugin build --target claude-code --output-dir dist/ci-plugin
      - name: Validate plugin
        run: claude plugin validate --strict dist/ci-plugin/claude-code
```

The `paths:` filter ensures the job only runs when plugin-related files change, keeping CI fast.

### T022 — Generate `marketplace.json` for git-based distribution

`marketplace.json` lives at the repository root (not inside the bundle dir). Generate it as part of `spec-kitty plugin build --target claude-code` or as a separate `spec-kitty plugin publish-metadata` command. The simpler approach: generate alongside the bundle.

Schema per `research.md` (git-subdir marketplace format — NOT the `plugin-manifest-claude.md` which documents the plugin bundle manifest, not the marketplace catalog):
```json
{
  "name": "spec-kitty-plugins",
  "interface": { "displayName": "Spec Kitty Plugins" },
  "plugins": [
    {
      "name": "spec-kitty",
      "source": {
        "source": "git-subdir",
        "url": "https://github.com/Priivacy-ai/spec-kitty.git",
        "path": "dist/spec-kitty-plugins/claude-code"
      },
      "policy": { "installation": "AVAILABLE", "authentication": "ON_INSTALL" },
      "category": "Developer Tools"
    }
  ]
}
```

Write to `<output_dir>/marketplace.json` alongside the bundle. Do NOT commit `marketplace.json` to the source repo — it is a build artifact. Add it to `.gitignore` if not already excluded.

### T023 — Add `docs/how-to/install-claude-code-plugin.md`

Create `docs/how-to/install-claude-code-plugin.md` covering two install paths:

**Marketplace install (3-minute setup):**
```bash
# Requires Claude Code CLI v2.0.12+
claude plugin marketplace add https://github.com/Priivacy-ai/spec-kitty
claude plugin list  # verify spec-kitty appears
```

**Dev install (for contributors):**
```bash
# From the spec-kitty repo root:
spec-kitty plugin build --target claude-code
claude --plugin-dir dist/spec-kitty-plugins/claude-code <your-prompt>
# Or for persistent dev use:
claude plugin install dist/spec-kitty-plugins/claude-code
```

Include a troubleshooting section:
- "Plugin agents not appearing in /agents" → run `claude plugin list` to verify install; try `claude plugin uninstall spec-kitty && claude plugin install <path>`
- "wrapper: uvx not found" → install uv: `curl -LsSf https://astral.sh/uv/install.sh | sh`

Keep the doc concise (under 100 lines). Reference `CONTRIBUTING.md` for build system details.
